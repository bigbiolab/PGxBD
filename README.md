# PGxBD

Pharmacogenomic (PGx) allele-frequency database for the Bangladeshi population, built from
1000 Genomes Project phase 3 data with a focus on the Bengali-in-Bangladesh (BEB) population
(N=86). Frequencies are computed for 29 key PGx variants across 7 populations, with star-allele
and phenotype frequencies derived under Hardy-Weinberg equilibrium. v1 scope: 1000 Genomes
BEB-vs-global comparison only (no literature curation, no PharmVar-linked star-allele
definitions — see `docs/major_limitations.md`).

## Key Features

- **29 PGx variants** across 18 pharmacogenes (CYP2C19, CYP2C9, CYP2D6, CYP3A5, CYP3A4, CYP2B6,
  TPMT, SLCO1B1, DPYD, VKORC1, MTHFR, CYP4F2, G6PD, F5, NUDT15)
- **7 populations**: BEB (N=86), SAS excl. BEB (N=403), SAS (N=489), EUR (N=503), EAS (N=504),
  AFR (N=661), AMR (N=347)
- Corrected allele orientation for strand-flipped genes (CYP3A5\*3, F5 Leiden)
- Full HWE phenotype computation including compound heterozygotes (CYP2C19 \*2/\*17, \*3/\*17)
  and decreased-function alleles (CYP2D6 \*10)
- SQLite database with foreign-key linkage + FastAPI REST API
- Validation suite with automated sanity checks


## Setup

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## Running the Pipeline

```bash
# Full pipeline (downloads VCFs + reference data, computes everything)
bash scripts/run_pipeline.sh

# Skip VCF download (use already-staged VCFs)
bash scripts/run_pipeline.sh --skip-fetch
```

WSL stages need `scripts/00_setup_wsl_env.sh` run once first (bootstraps pysam/cyvcf2 for
remote-tabix access to 1000 Genomes VCFs). Individual steps can also be run separately, e.g.:

```bash
python scripts/02_compute_frequencies.py
python scripts/04_build_database.py
python scripts/05_compute_phenotypes.py
python scripts/06_validate.py
python scripts/07_generate_figures.py
```

## Running the API

```bash
uvicorn api.main:app --reload --port 8000
```

API documentation is available at `http://localhost:8000/docs`.

## Running Tests

```bash
pytest tests/test_api.py -v
```

## Running the UI

```bash
cd web
npm install
npm run dev
```

Open `http://localhost:3000`. The UI reads `NEXT_PUBLIC_API_URL` from `web/.env.local`
(defaults to `http://localhost:8000`) — start the API first. Pages: Overview, Genes,
Frequencies, Phenotypes, Drugs, Variants (cross-population heatmap).

## Corrections Applied (through v2.0.0)

1. **CYP3A5\*3 allele orientation**: CYP3A5 is on the minus strand; the \*3 allele (rs776746)
   corresponds to the REF allele, not ALT. Corrected BEB \*3 frequency: 0.366 → 0.634.
2. **CYP3A4\*22 vs \*1G**: rs2242480 is CYP3A4\*1G, not \*22. The true \*22 variant is rs35599367
   (c.520-191C>T). Both are now correctly labeled.
3. **CYP2C19 compound heterozygotes**: \*2/\*17 and \*3/\*17 are classified Intermediate
   Metabolizer per CPIC; previously omitted from the HWE calculation. Corrected BEB
   Intermediate: 0.377 → 0.454.
4. **G6PD frequencies**: recomputed with correct genomic positions and chrX-aware (hemizygous
   male) counting. BEB AF for rs1050828/rs1050829: 0.244 → 0.0.
5. **DPYD \*2A vs C29R**: rs1801265 is C29R (c.85C>T), not \*2A. The true \*2A variant is
   rs3918290 (c.1905+1G>A). Both are now correctly labeled.
6. **CYP2D6 \*10**: decreased-function allele (rs1065852, BEB AF=0.256), previously ignored,
   is now included with activity score 0.25.
7. **SAS contains BEB**: BEB is a SAS subpopulation. Added SAS_EXCL_BEB (N=403) as an
   independent comparison population.

Full detail in `docs/major_limitations.md` and `docs/v2_correction_plan.md`.

## Data Sources

- **1000 Genomes Project** phase 3 (GRCh37): VCF files and sample panel
- **PharmGKB / CPIC**: gene/variant/clinical annotations, phenotype-to-genotype tables
- **PyPGx**: star-allele defining-variant catalog
- **dbSNP**: variant genomic coordinates (GRCh37)
